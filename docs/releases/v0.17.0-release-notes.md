# Routa Desktop v0.17.0

## Summary

# v0.17.0 AI Summary

## Highlights

- **Feature Explorer became the dominant new surface in this release.** The product now exposes a much richer feature-first information architecture across both web and Rust backends, including route-tree navigation, surface index browse modes, feature metrics, repo switching, session analysis drawers, related file lists, and learned prompt context summaries.

- **Session and trace analysis is materially stronger.** Feature tracing moved into dedicated crates and parsers, transcript/session discovery became more robust, and the explorer can now connect selected files, routes, surfaces, and prompt signals back to real sessions with better parity across runtimes.

- **Harness, fitness, and release operations were tightened.** This range includes graph-aware test mapping snapshots, pi-autoresearch-driven fitness speed optimization, release-skill preparation flow, shared repo-level skill conventions, and several validation, portability, and pre-push stability fixes.

- **Team, spec, canvas, and MCP surfaces all advanced in parallel.** Team run surfaces gained better Rust/backend parity, the Spec area evolved into a local issues visualization board with harness integration, and canvas/sdk resources became first-class MCP artifacts shared across runtimes.

## Notable Areas

- `feature-explorer`: major expansion across layout, IA, backend parity, feature-tree ingestion, session analysis, taxonomy inference, and learned prompt context
- `harness` / `entrix`: speed optimization, graph-aware mapping snapshots, validation follow-ups, and release preparation workflow
- `team`: backend team-run surface parity, cycle-safe descendant counts, localized unnamed-run handling, and lead-specialist markers
- `spec` / `canvas` / `mcp`: local issue visualization board, canvas SDK resources, constrained React runtime artifacts, and cross-runtime contract sharing

## Release Shape

This is a feature-heavy release rather than a patch train. Compared with `v0.16.0`, the scope is dominated by the Feature Explorer initiative, with supporting improvements in fitness automation, backend parity, issue/spec tooling, and release ergonomics.
